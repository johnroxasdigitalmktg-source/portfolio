AI Voice Agent Deployment

Real screenshots from a live client build

THE PROBLEM

Leads were filling out the intake form but not pushing through with booking an appointment. They would drop off at the scheduling step — interested enough to fill the form, but not enough to complete the booking on their own.

THE SOLUTION

A Retell AI voice agent is deployed to call the lead automatically. Instead of sending another email they'll ignore, the AI calls them directly — introduces itself, confirms their interest, and walks them through picking a slot over the phone. No forms. No friction. Just a conversation.

THE RESULT

The lead is booked in real-time directly into the GHL calendar. No staff involvement. Lead recovered, appointment confirmed, pipeline moves forward.

AGENT CONFIGURATION — RETELL AI BUILDER

The full agent setup inside Retell AI for the "Book Appointment Lead Recovery" agent. This includes a custom system prompt defining the agent's goal, tone, and call flow — scripted to never ask for information already in the CRM. Language rules handle multilingual calls (English & French) with automatic language detection mid-call. The knowledge base and custom functions are connected here. On the right, a live test chat shows the AI responding naturally in French, confirming it adapts to the caller's language in real-time.

CUSTOM API FUNCTIONS — HOW THE BOOKING ACTUALLY WORKS

Three custom-coded API functions built specifically for this agent. These are what make the AI capable of interacting with GHL — not just talking, but actually completing the booking:

  • get_slots-connection-retell-98b36f7e — The first function called in every booking flow. Queries GHL's calendar API and returns a list of available appointment slots based on the caller's resolved timezone. The AI reads these slots out loud to the caller and waits for a verbal confirmation before proceeding. Without this, the agent would have no real-time visibility into what's actually available — it would be guessing.
  • create_appointment_connection-retell — Fires only after the caller verbally confirms a slot from get_slots. Hits GHL's API endpoint to create the appointment in real-time. Takes an ISO8601 datetime with timezone offset (e.g. 2026-06-11T08:30:00-04:00). Never books a slot that wasn't explicitly confirmed by the caller.
  • timezone_resolver-custom-retell — Runs before get_slots to resolve the caller's timezone from their phone number or city + country. Uses ISO country codes. If unresolvable, falls back to America/Toronto. This step is critical — without it, slots would be returned in the wrong timezone and the appointment would land at the wrong time in GHL.
APPOINTMENT REPORTING — GHL DASHBOARD

Appointment reporting dashboard configured in GHL to track the impact of the AI voice agent — and the results speak for themselves. No-show rates dropped significantly once automated reminders and recovery sequences were in place. Total appointments, show rates, cancellations, and no-shows all tracked in one view, giving full visibility to keep optimizing what's already working.

LIVE PROOF — GHL CALENDAR

The GHL calendar showing John Jefferson Roxas — 08:30 AM to 09:00 AM, circled and annotated: "AI Voice Agent booked an appointment." This is a real booking from a real call. The AI identified the lead, resolved the timezone, pulled available slots from GHL, confirmed one with the caller, and fired the API to create the appointment — all within a single phone call, with zero human involvement.
